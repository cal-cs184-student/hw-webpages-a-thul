Overview

In this homework, I implemented de Casteljau's algorithm to evaluate Bezier curves and surfaces, as well as a mesh editor/upsampler with the capability to flip/split edges in a 3D mesh, as well as upsample to higher resolutions! I found the upsampling to be the most interesting part of the assignment, especially seeing its effect on performance in the renderer.

Section I: Bezier Curves and Surfaces

Part 1: Bezier Curves with 1D de Casteljau Subdivision (13 pts)

At each step, de Casteljau's algorithm works by finding points that are linearly interpolated between the endpoints of each line (parametrized by a variable t), then recursively applying this step between the computed points until there exists only one point left between the starting and ending control point. Then, the curve is drawn through these points.

I implemented a single step of this algorithm (so one set of interpolations, reducing the number of points from n to n-1) in the method evaluateStep, and this iterates through the input vector, calculates the interpolation between each successive points based on t, and then appends the resulting point to the output vector. This would then be recursively called n-1 times total until there is only one point left.

Part 2: Bezier Surfaces with Separable 1D de Casteljau (15 pts)

To extend de Casteljau's algorithm to bezier surfaces, we first evaluate the bezier curve along each row (interpolation parametrized by the variable u), condensing each one to a single point, and then, with these resulting points, evaluate the bezier curve along these points (interpolation parametrized by the variable v), resulting in one final point.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals (12 pts)

I implemented area weighted normals by iterating over a vertex's incident edges (via h = h->twin()->next) for each neighboring non-boundary face. For each face, I took the normal vector defined by the cross product of two of the vectors defining the face, giving a magnitude equivalent to the area of the parallelogram (2x the area of the triangular face) along a normal vector to that face, and took a running sum of these vectors. Then, at the end, I normalized the resulting vector and assigned it to the vertex. The running sum was already proportional to the sum of areas, so I didn't feel the need to divide by two partway.

Part 6: Loop Subdivision for Mesh Upsampling (25 pts)

I followed the provided comments for my overall architecture of loop subdivision, finding all new vertices via the loop subdivision rule, calculating new midpoints for splits via the 3/8-1/8 rule, then splitting all edges, and finally flipping new edges that connect new and old vertices. I'd say the main implementation note that I have is that I implemented marking edges as new/old using the fact that the edges a-m, m-b are along an existing edge a-b, so they are old. New edges connect c and d to m, so they are new! This is done during the split.

With sharp edges, the resulting mesh tends to smooth everything out. Since every edge is treated the same way, new vertex positions are weighted averages of their neighboring vertices, and as such, as seen on the upsampled cube below, upsampled vertices are pulled between existing vertices on different faces, rounding out the edges.

With the original meshing of the cube, different corner vertices had different degrees on each face, due to there only being a single diagonal. This would pull vertices in unequally, making the meshing asymmetric for each face. I split the diagonal edges (as well as another layer of splitting) such that each face has a symmetric mesh, and all vertices have the same degree, making upsampling weight them equally. This created a much more symmetric, albeit rounded, cube (at maximum upsampling).
